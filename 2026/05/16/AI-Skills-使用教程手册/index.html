3. 研究调研与选题类

3.1 deep-research

一句话用途

deep-research 用来做多来源联网深度调研，适合快速了解某个研究方向、技术路线、竞品格局或最新热点。

适合场景

  • 调研开放世界检测、开放词表检测等研究方向
  • 查某个领域最近 1-2 年热点
  • 做选题前的方向扫描
  • 写带来源链接的研究报告
  • 对比多个方法、模型、工具或数据集

前置条件

已配置 Exa MCP：

基本用法

Claude Code：

注意事项

deep-research 适合第一轮方向扫描，不建议把它的结论直接当论文依据。真正写论文前，要精读核心论文并核对引用。

3.2 research-ideation

一句话用途

research-ideation 用来把模糊想法收敛成可执行研究问题，核心方法是 5W1H、gap analysis 和 research question card。

适合场景

  • 想开题，但问题还不够具体
  • 想从大方向里找可发论文的小切口
  • 想整理 research gap、假设、证据需求和最小实验
  • 想判断一个想法是否可做、可证伪、可复现

基本用法

Claude Code：

输出重点

优先看它生成的 Research Question Card：

3.3 literature-review

一句话用途

literature-review 用来做系统综述、scoping review、meta-analysis 或大规模文献综述草稿。

适合场景

  • 写毕业论文 related work 大章
  • 做开放世界检测方向系统综述
  • 做方法分类、数据集分类和 benchmark 梳理
  • 需要明确检索范围、纳入/排除标准和引用格式

基本用法

注意事项

这个 skill 比 deep-research 更正式、更重。普通快速调研优先用 deep-research；需要写综述或论文 related work 大章时再用它。

3.4 explaining-academic-papers

一句话用途

explaining-academic-papers 用来精读论文，并生成多文件 Markdown 笔记：总览、逐节精读、图表讲解、思维导图。

适合场景

  • 精读一篇核心论文
  • 把论文和代码仓库对应起来
  • 做组会讲解前的系统笔记
  • 梳理论文方法、公式、实验和指标

基本用法

注意事项

这个 skill 会生成文件，默认适合比较完整的论文精读任务。如果只是问“这篇论文讲了什么”，可以先用轻量总结。

4. AMiner 学术数据类

4.1 aminer-data-search

一句话用途

aminer-data-search 是 AMiner 学术数据检索总入口，会根据任务自动路由到免费检索、深度检索或论文推荐。

适合场景

  • 查论文
  • 查学者
  • 查机构
  • 查期刊/会议
  • 查专利
  • 查引用链
  • 查某个方向代表论文和代表学者

AMiner 深度接口可能收费。默认应先走免费接口；预计费用超过 5 元时要先确认。

4.2 aminer-free-academic

一句话用途

aminer-free-academic 只使用 AMiner 免费接口，适合轻量检索和初筛。

适合场景

  • 已知论文标题，查 paper id、年份、venue
  • 查某位学者是否是目标人
  • 查机构规范名称
  • 查期刊/会议类型
  • 先判断一个方向是否值得深挖

基本用法

注意事项

免费接口字段有限。需要完整摘要、引用链、机构产出分析时，转 aminer-academic-search。

4.3 aminer-academic-search

一句话用途

aminer-academic-search 是 AMiner 深度学术分析 skill，覆盖论文详情、引用关系、学者画像、机构分析、期刊监控和专利分析。

适合场景

  • 深入分析一篇论文
  • 查某位学者完整画像
  • 查机构科研能力
  • 查某个 venue 某年的论文
  • 查论文引用链和相关论文

基本用法

注意事项

可能调用付费接口。高成本任务必须先确认。

4.4 aminer-daily-paper

一句话用途

aminer-daily-paper 用于按主题、学者或 AMiner 账号做个性化论文推荐。

适合场景

  • 每天看几篇方向相关论文
  • 按研究主题推荐论文
  • 按学者画像推荐论文
  • 跟踪开放世界检测、开放词表检测、多模态检测等方向

基本用法

注意事项

中文主题可以直接保留中文，不要强行翻译成别的研究方向。

5. 引用、文献与写作类

5.1 citation-management

一句话用途

citation-management 用来查论文元数据、生成 BibTeX、清理 .bib、去重和统一引用格式。

适合场景

  • DOI / arXiv / PMID 转 BibTeX
  • 清理 references.bib
  • 查重和统一引用 key
  • 检查作者、标题、年份、venue 是否正确
  • 投论文前整理参考文献

基本用法

Claude Code：

注意事项

不要让 AI 直接编 BibTeX。优先用 DOI、arXiv、CrossRef、Semantic Scholar 等来源核对。

5.2 citation-verification

一句话用途

citation-verification 用来核验引用真实性、DOI、作者、年份、venue 和“引用是否真的支撑对应句子”。

适合场景

  • 防止假引用
  • 检查 AI 生成的引用
  • 检查 related work 里的引用是否支持原句
  • 检查 .bib 里 DOI、作者、年份是否错误
  • 投稿前引用体检

基本用法

Claude Code：

注意事项

它和 citation-management 很互补：一个负责管理格式，一个负责核验真实性和支撑关系。

5.3 nature-polishing

一句话用途

nature-polishing 用于 Nature 风格学术英文润色、中译英、摘要/引言/结果/讨论重写。

适合场景

  • 中文论文段落转英文
  • 英文摘要润色
  • 引言逻辑重构
  • 结果段落降噪
  • 讨论部分降低 overclaim

基本用法

注意事项

不要让它凭空生成论文核心结论。它适合改表达、改逻辑，不适合发明研究贡献。

5.4 nature-citation

一句话用途

nature-citation 用来给句子或段落寻找 Nature/CNS/Cell/Science 方向支撑引用，并可导出 RIS/ENW/Zotero RDF。

适合场景

  • 给引言补引用
  • 给讨论补背景文献
  • 按 claim 查支撑论文
  • 判断文献是强支撑、部分支撑还是背景支撑

基本用法

注意事项

题目相关不代表能支撑原句。最终仍要人工核对摘要、实验和结论。

5.5 nature-data

一句话用途

nature-data 用来写 Data Availability、数据仓库计划、FAIR 元数据检查和数据引用。

适合场景

  • Nature 系列投稿的数据可用性声明
  • 数据是否公开、受限、请求获取
  • 补充材料、source data、代码公开说明
  • FAIR checklist

基本用法

注意事项

不要笼统写 “available upon reasonable request”。需要说明数据类型、限制原因、获取流程和责任人。

5.6 nature-response

一句话用途

nature-response 用来写审稿意见逐点回复、rebuttal letter 和修回信。

适合场景

  • 大修/小修回复
  • 审稿意见分类
  • 回复草稿降火
  • 将每条意见映射到具体修改

基本用法

注意事项

不要让它编造已经完成的实验或修改位置。缺信息时必须标 AUTHOR_INPUT_NEEDED。

6. 图表、PPT 与网页工具类

6.1 nature-figure

一句话用途

nature-figure 用来生成 Nature / 高水平期刊风格科研图、多面板图和投稿级 SVG/PNG/PDF/TIFF。

适合场景

  • 多面板科研图
  • 柱状图、折线图、热图、散点图、气泡图
  • 统一字体、颜色、图例和 panel label
  • 输出可编辑 SVG

基本用法

6.2 nature-paper2ppt

一句话用途

nature-paper2ppt 用来把论文、PDF、摘要或阅读笔记做成中文组会/文献汇报 PPTX。

适合场景

  • 组会汇报
  • journal club
  • 论文分享
  • 课程展示
  • 论文答辩前的背景介绍

基本用法

注意事项

如果只给标题和摘要，不要让它虚构具体实验图。缺失内容要标占位。

6.3 ppt-image-to-editable-ppt

一句话用途

ppt-image-to-editable-ppt 用来把 PPT 截图或导出的 slide 图片重建成可编辑 PowerPoint。

适合场景

  • 只有 PPT 图片，没有原始 .pptx
  • 想把截图里的文字变成可编辑文本框
  • 想把图标、logo、照片拆成独立 PNG
  • 想重建卡片、线条、圆形、分隔符等 PPT 原生形状

基本用法

注意事项

不要把整张截图直接铺成背景图。那样虽然看起来像，但不可编辑。

6.4 defuddle

一句话用途

defuddle 用来把网页正文清洗成干净 Markdown，去掉导航栏、广告、页脚和杂乱 HTML。

适合场景

  • 读取项目主页
  • 清洗在线文档
  • 把博客/教程转成 Markdown
  • 降低网页噪声，方便后续总结和写作

9. Codex 内置/系统 Skill 简表

下面这些是 Codex 自带或插件提供的系统级 skill，通常不需要手动安装，也不一定会同步成 Claude Code slash command。日常使用时，直接用自然语言触发即可。

9.1 imagegen

一句话用途

imagegen 用来生成或编辑位图图片，比如封面图、插图、纹理、素材图和透明背景 cutout。

适合场景

  • 给博客文章生成封面图
  • 给 PPT 做示意图素材
  • 生成论文汇报里的非数据型插图
  • 根据参考图生成风格接近的视觉素材

基本用法

注意事项

科研数据图优先用 nature-figure，不要用生成图替代真实实验结果。

9.2 openai-docs

一句话用途

openai-docs 用来查询 OpenAI 官方文档、模型能力、API 用法和迁移建议。

适合场景

  • 写 OpenAI API 代码前查最新参数
  • 比较当前可用模型
  • 检查 Responses API、工具调用、结构化输出等用法
  • 升级旧代码里的模型或接口

基本用法

注意事项

涉及 OpenAI 产品时优先查官方文档，不要只凭记忆写代码。

9.3 skill-installer

一句话用途

skill-installer 用来安装 Codex skill，可以从 curated list 或 GitHub 仓库安装。

适合场景

  • 安装别人分享的 skill
  • 更新本地 skill
  • 检查某个 skill 是否已经安装
  • 把 GitHub 上的 skill 装进 Codex

基本用法

注意事项

安装后通常要重启 Codex 或重新打开会话，新的 skill 才会进入可用列表。

9.4 skill-creator

一句话用途

skill-creator 用来创建或改造自己的 skill，把固定工作流沉淀成可复用能力。

适合场景

  • 把自己的论文精读流程做成 skill
  • 把某类代码审查规范做成 skill
  • 把实验记录、复现检查、论文润色模板固化下来
  • 给团队共享统一工作方式

基本用法

注意事项

好的 skill 应该写清触发场景、输入约定、输出格式和质量检查，不要只写一段泛泛提示词。

9.5 plugin-creator

一句话用途

plugin-creator 用来创建 Codex plugin 目录结构，包括 plugin.json、skills、MCP 或应用入口等。

适合场景

  • 想把多个 skill 和 MCP 打包成一个插件
  • 想做一个本地 Codex 插件模板
  • 想把团队工具链整理成可分发 plugin

基本用法

注意事项

普通个人工作流通常先做 skill；只有多个能力需要打包、共享或带 MCP 时，再考虑 plugin。

9.6 chrome

一句话用途

chrome 是 Chrome 浏览器自动化能力，用来操作需要登录态、Cookie、扩展或现有标签页的网站。

适合场景

  • 打开需要登录的学术平台
  • 操作网页后台或表单
  • 读取当前 Chrome 标签页内容
  • 配合浏览器扩展完成需要用户会话的任务

基本用法

注意事项

普通公开网页检索优先用 deep-research 或浏览工具；只有需要你的浏览器登录态时，再用 chrome。
