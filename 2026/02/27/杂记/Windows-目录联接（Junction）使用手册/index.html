1. 概念介绍

目录联接（Junction） 是 Windows NTFS 文件系统提供的一种特殊链接方式。它可以让一个目录"指向"另一个目录，使得访问链接目录时，实际上访问的是目标目录中的内容。

通俗理解

想象你在家里的书架 A 上放了一个"传送门"，走进去就会到达书架 B。书架 B 的书没有被移动，但你可以通过书架 A 来读取它们。

核心特点

特点 说明
不复制数据 只创建一个"指针"，不占用额外磁盘空间
实时同步 通过链接路径和原始路径访问的是同一份数据
不需要管理员权限 普通用户即可创建
仅支持本地路径 不能指向网络路径
仅支持目录 不能用于单个文件（文件需用硬链接或符号链接）

2. 前置准备

2.1 打开命令提示符（CMD）

[!IMPORTANT] 以下命令需要在 CMD（命令提示符） 中执行，不是 PowerShell。

打开方式：

  • 按 Win + R，输入 cmd，回车
  • 或在开始菜单搜索"命令提示符"

2.2 确认路径

在操作前，请确认：

  • ✅ 目标目录（已存在的、想要被链接的文件夹）路径正确
  • ✅ 链接目录（即将创建的新入口）路径 不存在（不能已有同名文件夹）
  • ✅ 两者都在 本地 NTFS 磁盘 上

5. 解除（删除）联接

[!CAUTION] 这是最关键的部分，操作不当可能导致数据丢失！请仔细阅读。

5.1 ✅ 正确方法：使用 rmdir

效果：

  • ✅ 只删除链接入口本身
  • ✅ 不会 删除目标文件夹中的任何文件
  • ✅ 原始数据 C:\blog\source\_posts 完好无损

5.2 ❌ 危险方法：千万不要这样做

危险操作 后果
在资源管理器中直接 Delete 或 Shift+Delete ⚠️ 可能删除目标文件夹中的真实文件！
使用 del /S /Q "链接路径" ⚠️ 会删除目标文件夹中的所有文件！
使用 rd /S /Q "链接路径" ⚠️ 会递归删除目标文件夹中的所有内容！

[!WARNING] 切记： rmdir "链接路径"（不带 /S）是唯一安全的删除方式！

带 /S 参数的 rmdir（即 rd /S）会穿透链接删除真实文件，绝对不要使用！

6. 注意事项与安全守则

6.1 日常使用注意

  1. 读写数据：通过链接路径和原始路径操作的是同一份数据，在任何一边修改都会反映到另一边
  2. 移动目标文件夹：如果移动或重命名了目标文件夹，联接会失效（变成"死链接"），需要删除旧联接并重新创建
  3. 磁盘格式：联接仅在 NTFS 文件系统上有效，FAT32/exFAT 不支持

6.2 云同步场景特别注意

事项 建议
云盘客户端兼容性 大多数云盘客户端（小米云盘、OneDrive、坚果云等）支持同步联接目录中的内容
双向同步冲突 如果同一文件在多个设备上被修改，可能产生冲突文件
首次同步 创建联接后，首次同步可能需要较长时间上传所有文件
同步删除 如果在云端或其他设备上删除了文件，本地联接目标中的对应文件也会被云盘客户端删除

8. 附录：Junction vs Symbolic Link 对比

特性 Junction（目录联接） Symbolic Link（符号链接）
创建命令 mklink /J mklink /D
需要管理员权限 ❌ 不需要 ✅ 需要（或开启开发者模式）
支持目标类型 仅本地目录 本地/网络目录和文件
跨驱动器 ✅ 支持 ✅ 支持
支持相对路径目标 ❌ 始终存储绝对路径 ✅ 支持
远程网络路径 ❌ 不支持 ✅ 支持
推荐场景 本地目录链接（如云盘同步） 需要链接文件或网络路径时

[!TIP] 对于本地目录的同步需求，Junction 是最简单、最安全的选择。 不需要管理员权限，兼容性好，性能无损。
