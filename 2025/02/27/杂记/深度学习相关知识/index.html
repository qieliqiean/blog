Batch Normalization（批量归一化） 和 Layer Normalization（层归一化）

1. 核心概念对比

Batch Normalization (BN)

  • 归一化方向：对 同一特征维度 跨所有样本和空间位置归一化。

  • 核心公式：

    假设输入张量形状为 (N, C, H, W)（Batch Size, Channels, Height, Width），则对每个通道 c 计算均值和方差：

    μc=1N⋅H⋅W∑n=1N∑h=1H∑w=1Wxn,c,h,w,σc2=1N⋅H⋅W∑n=1N∑h=1H∑w=1W(xn,c,h,w−μc)2

  • 适用场景：图像处理（CNN）、固定长度的结构化数据。

Layer Normalization (LN)

  • 归一化方向：对 同一样本 的所有特征维度归一化。

  • 核心公式：

    假设输入张量形状为 (N, C, H, W)，对每个样本 n 计算均值和方差：

    μn=1C⋅H⋅W∑c=1C∑h=1H∑w=1Wxn,c,h,w,σn2=1C⋅H⋅W∑c=1C∑h=1H∑w=1W(xn,c,h,w−μn)2

  • 适用场景：序列数据（RNN、Transformer）、动态长度输入。

2. 公式符号解释

输入张量形状

假设输入为 (N, C, H, W)：

  • N: 批次大小（Batch Size）
  • C: 通道数（特征维度）
  • H: 高度（空间维度）
  • W: 宽度（空间维度）

BN 公式符号

  • μₐ: 第 c 个通道的均值（跨所有样本和空间位置）。
  • σₐ²: 第 c 个通道的方差。
  • xₙ,ₐ,ₕ,ᵥ: 第 n 个样本、第 c 个通道、位置 (h, w) 的像素值。

LN 公式符号

  • μₙ: 第 n 个样本的均值（跨所有特征和空间位置）。
  • σₙ²: 第 n 个样本的方差。
  • xₙ,ₐ,ₕ,ᵥ: 同上。

3. 具体示例对比

输入数据

  • N=2（2张图像），C=3（RGB三通道），H=2, W=2（2x2像素）。
  • 样本1：
    • 通道0（红）：[[1, 3], [5, 7]]
    • 通道1（绿）：[[2, 4], [6, 8]]
    • 通道2（蓝）：[[3, 5], [7, 9]]
  • 样本2：
    • 通道0（红）：[[2, 4], [6, 8]]
    • 通道1（绿）：[[3, 5], [7, 9]]
    • 通道2（蓝）：[[4, 6], [8, 10]]

BN 计算示例（以通道0为例）

  1. 计算均值 μ₀： μ0=(1+3+5+7)+(2+4+6+8)8=16+208=4.5
  2. 计算方差 σ₀²： σ02=(1−4.5)2+(3−4.5)2+⋯+(8−4.5)28=21+218=5.25
  3. 归一化后：每个像素值变为 (x - 4.5) / √(5.25 + ε)。

LN 计算示例（以样本1为例）

  1. 计算均值 μ₁： μ1=1+3+5+7+2+4+6+8+3+5+7+912=6012=5
  2. 计算方差 σ₁²： σ12=(1−5)2+(3−5)2+⋯+(9−5)212=8412=7
  3. 归一化后：每个像素值变为 (x - 5) / √(7 + ε)。

4. 对比总结表格

特性 Batch Normalization (BN) Layer Normalization (LN)
归一化维度 同一特征维度跨样本和空间位置 同一样本的所有特征维度
参数数量 2C（每通道独立参数） 2（全局共享参数）
依赖条件 需大 Batch Size（如32+） 不依赖 Batch Size
适用场景 图像（CNN）、静态数据 文本（Transformer）、动态序列数据
训练/测试差异 测试时用全局移动平均统计量 测试时直接计算当前样本统计量

5. 注意事项

  1. BN 的局限性：

    • 小 Batch Size 时性能下降（方差估计不准确）。
    • 对序列数据（如变长文本）不友好。

  2. LN 的优势：

    • 适用于单样本推理（如实时生成文本）。
    • 对特征维度间的动态变化更鲁棒。

上采样（Upsampling）与下采样（Downsampling）详解

核心区别

  • 上采样：放大特征图尺寸，恢复细节信息（如目标边缘、小物体位置）。
  • 下采样：缩小特征图尺寸，提取抽象语义（如目标类别、整体形状）。
  • 协作关系：两者结合实现多尺度检测（如同时识别近处大车和远处小车）。

1. 下采样（Downsampling）

作用

  • 压缩特征图，减少计算量。
  • 捕获高层语义信息（例如“这是一辆车”）。

实现方式

方法 说明
跨步卷积 增大卷积步长（如stride=2），跳过部分像素，直接缩小特征图尺寸。
池化（Pooling） 取局部区域的最大值（Max Pooling）或平均值（Average Pooling）。

特点

  • 输出尺寸：缩小（例如输入256x256 → 输出128x128）。
  • 信息保留：丢弃细节，保留整体特征。
  • 类比：眯眼看照片，忽略纹理细节，只关注轮廓。

2. 上采样（Upsampling）

作用

  • 扩大特征图尺寸，辅助精确定位（如车轮、车灯位置）。
  • 与浅层特征融合，增强小目标检测能力。

实现方式

方法 说明
插值（Interpolation） 通过双线性插值、最近邻插值等算法填充像素（类似图像放大）。
转置卷积（Transposed Convolution） 反向卷积操作，通过可学习参数生成更大特征图。

特点

  • 输出尺寸：放大（例如输入128x128 → 输出256x256）。
  • 信息保留：补充细节，但可能引入噪声。
  • 类比：放大模糊图片，猜测缺失的像素。

对比总结

特性 下采样 上采样
目的 提取抽象特征，减少计算量 恢复细节，精确定位
典型操作 池化、跨步卷积 插值、转置卷积
信息流 从细节到语义 从语义到细节
应用阶段 Backbone（特征提取层） Neck（特征融合层）

实际例子

  • 场景：检测一张包含“近处行人”和“远处车辆”的图像。
  • 下采样：深层特征图（20x20）捕获“远处车辆”的抽象轮廓。
  • 上采样：将该特征图放大到40x40，与浅层特征（包含“近处行人”细节）融合，实现多尺度检测。

空洞卷积（Dilated Convolution）详解

1. 概述

空洞卷积（Dilated Convolution），又称膨胀卷积或扩张卷积，是一种通过插入“空洞”（间隔）来扩大感受野的卷积操作。它能在不增加参数量的前提下捕捉更大范围的上下文信息，常用于图像分割、语音合成等任务。

2. 为什么需要空洞卷积？

问题背景：

  • 传统卷积神经网络（CNN）通过堆叠卷积层和池化层（下采样）逐步扩大感受野，但池化会降低图像分辨率（丢失细节），导致小目标（如红外弱小目标）难以识别。
  • 如果直接去掉池化层（降低下采样率），后续层的感受野会变小，模型无法捕捉全局特征，导致性能下降。

空洞卷积的解决方案：

  • 保持分辨率：不进行下采样（保留细节）。

  • 扩大感受野：通过空洞卷积核覆盖更大区域，避免池化带来的信息损失。

    类型 卷积核 感受野（3层后） 分辨率 适用场景
    普通卷积（3×3） 连续像素 7×7 逐渐降低（需池化） 常规目标检测
    空洞卷积（3×3，rate=2） 间隔1像素 15×15 保持原始分辨率 小目标检测、图像分割

3. 优点

优势 说明
更大的感受野 捕捉长距离依赖，适合分割/检测任务
保持分辨率 无需下采样即可扩大感受野，减少信息损失
参数量不变 相比堆叠卷积层或增大核尺寸更高效

4. 缺点

缺点 说明
网格效应（Gridding） 高扩张率可能导致局部信息丢失
优化难度 需平衡扩张率与任务需求

MIRST与SIRST

SIRST

  • 全称 (Full Name): Single-frame Infrared Small Target (单帧红外小目标)
  • 含义 (Meaning): SIRST问题指的是在单张红外图像（即一张静态照片或视频中的某一帧）中检测出小型和暗弱目标的问题。
  • 特点 (Characteristics):
    • 局限于空间信息： 算法主要依赖于目标与背景在空间域（如亮度、对比度、形状、纹理等）上的差异来识别目标。
    • 不利用时间信息： 由于只处理单帧，无法利用目标在不同时间帧之间的运动轨迹、背景变化等时间域信息。
    • 挑战： 如果目标的热对比度非常低，或者背景杂波复杂且与目标特征相似，单帧检测会非常困难，因为缺乏时间维度的辅助信息来区分目标和噪声/杂波。
  • 相关算法： 许多传统的基于图像处理的方法（如基于滤波、形态学、局部对比度等）以及一些早期的深度学习方法，都首先在SIRST问题上进行研究和验证。

MIRST

  • 全称 (Full Name): Multi-frame Infrared Small Target (多帧红外小目标)
  • 含义 (Meaning): MIRST问题指的是在连续的多帧红外图像序列（即视频流）中检测出小型和暗弱目标的问题。
  • 特点 (Characteristics):
    • 结合空间和时间信息： 除了单帧图像中的空间特征外，MIRST算法还能利用目标在不同帧之间的运动信息（如速度、方向、轨迹一致性）来辅助检测。
    • 区分运动目标与静止杂波： 这是MIRST最大的优势。真正的目标通常是运动的，而背景杂波、噪声或热点是相对静止或随机变化的。通过分析时间上的连续性，可以有效抑制静止杂波和瞬时噪声。
    • 挑战： 需要处理时间同步、运动估计、数据量大、计算复杂度高等问题。
  • 相关算法： 通常涉及背景抑制、目标轨迹跟踪、运动补偿、时空滤波等技术，近年来的深度学习方法也开始更多地利用视频序列信息来提升检测性能。

评估指标

TP（True Positives)：真正例，预测为正例而且实际上也是正例；

FP（False Positives)：假正例，预测为正例然而实际上却是负例；

FN（false Negatives)：假负例，预测为负例然而实际上却是正例；

TN（True Negatives)：真负例，预测为负例而且实际上也是负例。

检测率

Probability of Detection, Pd

  • 定义： 算法正确检测到的真实目标数，占所有真实目标数的比例。衡量 “不漏检” 能力，军事场景中 Pd 至关重要（如导弹告警，漏检会导致防御失效）。

  • 计算公式：

    Pd=True Positives (TP)True Positives (TP)+False Negatives (FN)=TPActual Positives
    • True Positives (TP，真阳性): 算法正确地将目标识别为目标（即检测到了真实存在的目标）。
    • False Negatives (FN，假阴性): 算法未能检测到真实存在的目标（即漏报了目标）。
    • Actual Positives (实际正例): 数据集中所有真实存在的目标数量 ( TP+FN )。

  • 含义与解读：

    • Pd 越高越好，表示算法的“捕获能力”越强，漏掉真实目标的可能性越小。
    • 在军事侦察、搜救等应用中， Pd 是一个非常关键的指标，因为它直接关系到任务的成功率和安全性。

虚警率

Probability of False Alarm, Pfa 或 Fa

  • 定义： 指的是检测算法错误地将背景或非目标物体识别为目标的比例。换句话说，算法报告的所有检测结果中，有多少是错误的。

  • 计算公式：

    Pfa=False Positives (FP)False Positives (FP)+True Negatives (TN)=FPActual Negatives
    • False Positives (FP，假阳性): 算法错误地将非目标识别为目标（即误报了）。
    • True Negatives (TN，真阴性): 算法正确地将非目标识别为非目标。
    • Actual Negatives (实际负例): 数据集中所有真实存在的非目标数量 ( FP+TN )。

  • 含义与解读：

    • Pfa 越低越好，表示算法的“识别准确性”越高，误报的可能性越小。
    • 过高的虚警率会导致大量无效警报，增加人工复核的负担，降低系统的实用性。

精确率

Precision

  • 定义：以预测结果为判断依据，预测为正例的样本中预测正确的比例。预测为正例的结果分两种，要么实际是正例TP，要么实际是负例FP

  • 计算公式：

    Precision=TPTP+FP

  • 含义与解读： 关注的是“查得准不准”。高精确率意味着算法报告的检测结果中，错误警报的比例较低。精确度还有一个名字，叫做**“查准率”，我们关心的主要部分是正例，所以查准率就是相对正例的预测结果而言，正例预测的准确度。直白的意思就是模型预测为正例的样本中，其中真正的正例占预测为正例样本的比例，用此标准来评估预测正例的准确度**。

召回率

Recall

  • 定义： 召回率就是检测率 Pd ，只是在机器学习领域更常使用“召回率”这个术语。

  • 计算公式： 同 Pd

    Recall=TPTP+FN

  • 含义与解读： 关注的是“查得全不全”。高召回率意味着算法能够找到大部分真实目标，很少有漏报。

F1 分数

F1-score

  • 定义： 精确率和召回率的调和平均值。当需要同时考虑精确率和召回率时，F1分数是一个很好的综合指标。

  • 计算公式：

    F1-score=2×Precision×RecallPrecision+Recall

  • 含义与解读： F1分数越高，表示算法在精确率和召回率之间取得了更好的平衡。

ROC 曲线

Receiver Operating Characteristic Curve

  • 定义： 以 Pfa （或 False Positive Rate, FPR，即虚警率）为横轴，以 Pd （或 True Positive Rate, TPR，即召回率）为纵轴绘制的曲线。通过改变分类阈值（例如，检测置信度），可以得到一系列的 ( Pfa,Pd ) 点。
  • 含义与解读：
    • ROC 曲线能直观地展示算法在不同检测阈值下， Pd 和 Pfa 之间的权衡关系。
    • 曲线越靠近左上角（即高 Pd 和低 Pfa ），表示算法性能越好。
    • 一条完美的检测算法会通过 (0,1) 点。
    • 对角线表示随机猜测的性能。

AUC

(Area Under the ROC Curve)

  • 定义： ROC 曲线下方的面积。

  • 含义与解读：

    • AUC 值介于 0 到 1 之间。
    • AUC 越高，表示算法的整体性能越好，因为它综合考虑了所有可能的阈值下的 Pd 和 Pfa 。
    • AUC 为 1 表示完美分类器；AUC 为 0.5 表示随机分类器。

    8. 准确率 Accuracy

    准确度：正例和负例中预测正确数量占总数量的比例。

    用公式表示：

    ACC=TP+TNTP+FP+FN+TN

在红外小目标检测中的特殊考虑：

在红外小目标检测中，通常会非常关注 Pd 和 Pfa 。因为小目标往往数量稀少、特征不明显，背景复杂，检测难度大。

  • 高 Pd 意味着能有效发现目标，避免漏报；
  • 低 Pfa 意味着系统不会被大量虚假警报淹没。

有时，研究者会绘制 Pd vs Pfa 曲线，以直观地比较不同算法在不同操作点（即不同的阈值设置）下的性能表现。这种曲线在目标检测领域通常被称为 Detection Error Tradeoff (DET) curve 或直接就是 ROC 曲线。

少样本目标检测

1.1 名词概念

  • 少样本对象检测 (Few-Shot Object Detection, FSOD): 每个新类别提供少量（比如1到10个）标注样本 。这是最主流的研究方向。

  • 单样本对象定位 (One-Shot Object Localization, OSOL): 这是 FSOD 的一个特例，每个新类别只提供一个标注样本 。

  • 零样本对象检测 (Zero-Shot Object Detection, ZSOD): 这是最极端的情况，新类别没有任何标注的图像样本 。那模型怎么学呢？它依赖于额外的信息，比如描述这些类别的语义属性（例如，描述“斑马”的词是“条纹”、“像马的动物”）

开放词汇检测

相关概念

1. 传统闭集检测 (Closed-Set Object Detection)

为了作为对比基准，我们先明确传统检测。

  • 概念： 训练集有什么类别，测试集就只能检测什么类别。
  • 表现： 模型就像一个死记硬背的学生。如果训练时只有“猫”和“狗”，测试时遇到“老虎”，模型要么将其误认为猫/狗，要么直接作为背景忽略。

2. 零样本目标检测 (Zero-Shot Object Detection, ZSD)

这是最早为了解决“未见类别（Unseen Classes）”而提出的范式。

  • 核心目标： 让模型在推理时，能够检测出在训练阶段从未提供过边界框和图像的类别。
  • 工作原理： 依赖于语义信息（Semantic Information）。模型在训练时，除了学习图像特征，还会学习类别名称的词向量（如 Word2Vec、GloVe）或人为定义的属性（如“有条纹”、“四条腿”）。测试时，模型通过对比图像特征和未见类别的语义向量，来实现跨模态的“联想”。
  • 局限性： ZSD 的测试类别通常是在测试前就已经“内定”好的一组固定词汇。它对语义映射的依赖很强，且泛化能力有限。
  • 一个关键细分：Generalized ZSD（GZSD）
    • ZSD：测试只考 unseen。
    • GZSD：测试同时包含 seen + unseen，更接近真实部署，也更难（模型偏向 seen）。（这是 ZSL/ZSD 社区常用划分，很多 ZSD 论文都会讨论这个“广义设定”。）

3. 开放词汇目标检测 (Open-Vocabulary Object Detection, OVD)

可以把它理解为 ZSD 在大模型时代的“究极进化版”。

  • 核心目标： 给定任意的文本提示（Text Prompt），模型都能在图像中定位并识别出对应的目标，即便这个词汇在训练集中从未出现过。
  • 工作原理： 深度依赖视觉-语言预训练大模型（VLMs），如 CLIP 或 ALIGN。模型将文本和图像投影到同一个高维特征空间中。因为语言大模型见过海量的互联网文本，OVD 模型具备极强的常识和泛化能力。
  • 与 ZSD 的核心区别： ZSD 通常只能处理单一的类别名称或预设属性。
    • OVD 能够理解自由形式的自然语言（Free-form text），词汇量是“开放”的。比如你可以直接输入“戴着红色帽子的泰迪熊”，OVD 模型也能尝试去检测它。

4. 开放世界目标检测 (Open-World Object Detection, OWOD)

这个概念最容易和 OVD 混淆，但它的侧重点完全不同。OWOD 关注的是动态发现与持续学习。

  • 核心目标： 模型不仅要识别已知类别，还要能主动发现未知对象并将其标记为“Unknown（未知）”，随后在人工介入标注后，能够**增量学习（Incremental Learning）**这些新类别，且不遗忘旧知识。
  • 工作原理： 包含两个核心机制：
    1. 未知目标提取器： 通过能量模型或伪标签技术，把不属于已知类别的显著性物体框出来，归类为“未知”。
    2. 持续学习机制： 当积攒了一批“未知”目标并获得真实标签后，模型只用很少的代价去微调，掌握新类别，同时避免灾难性遗忘（Catastrophic Forgetting）。
  • 与 OVD/ZSD 的核心区别： OVD 和 ZSD 在测试时，目标是“立刻认出”没见过的东西（因为有文本辅助）。而 OWOD 在面对没见过的东西时，它的反应是：“我不知道这是什么，但我知道它是一个实体（Unknown），等以后有人教我了，我就能记住它”。

5. 开集目标检测 (Open-Set Object Detection, OSOD)

  • 核心目标： 在测试时，模型不仅要准确识别并定位训练集中见过的已知类别（Known Classes），还要能够敏锐地发现没见过的新物体，并给它们统一打上**“未知（Unknown）”的标签。它的核心诉求是“不乱认”**——绝不能把未知的物体强行误认为已知类别，也不能简单地把它们当成背景忽略掉。
  • 工作原理： OSOD 往往依赖于**异常检测（Anomaly Detection）或不确定性估计（Uncertainty Estimation）**技术。在提取出候选框后，模型会去评估这个物体的特征分布。如果这个特征既不像背景，也不属于任何一个已知类别的特征分布，模型就会以较高的“不确定性”判定它为“未知”。
  • 与 OWOD (开放世界) 的核心区别： 还记得前面提到的 OWOD 吗？OWOD 包含两步：1. 发现未知；2. 增量学习记住它。
    • OSOD 只有第 1 步，没有第 2 步。 换句话说，OWOD = OSOD + 增量学习。OSOD 发现“未知”目标后任务就结束了，它并不会在后续去学习、更新模型来掌握这个新物体。
  • 与 OVD/ZSD 的核心区别：
    • OVD 和 ZSD 在面对没见过的东西时，会努力**“叫出它的名字”**（依赖文本提示或语义词典）。
    • OSOD 在面对没见过的东西时，只会诚实地回答：“这是一个物体，但我不知道它叫什么（Unknown）。” 它完全不需要文本或语言模型的辅助。
