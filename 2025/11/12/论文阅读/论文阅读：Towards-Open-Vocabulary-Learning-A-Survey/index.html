论文阅读：Towards Open Vocabulary Learning: A Survey

发表于2025-11-12|更新于2026-01-17|学习

1 INTRODUCTION

一、研究动机与现实困境

作者一开篇就指出，深度神经网络在视觉场景理解中取得了突破性进展——从目标检测（Object Detection）、图像分割（Segmentation）到目标跟踪（Tracking）都能取得极高精度。然而，传统方法普遍依赖“封闭类集合（Closed-set assumption）”：也就是说，模型只能识别训练集中出现过的预定义类别。

现实世界显然更复杂。以目标检测为例，主流数据集如 COCO 只有 80 个类别，而真实场景中往往包含成百上千种对象（家具、动植物、标志等）。若要让模型识别所有这些类别，就必须付出巨大的人工标注代价——既要画出每个目标的框（bounding box），又要给出类别标签。这一点在工业落地时尤其致命。

因此，作者提出一个核心现实问题：

如何让模型识别和理解“未见过的类别（novel classes）”，而不需要重新标注或重新训练？

二、从零样本学习（ZSL）到开放词汇（OVL）

为解决标注瓶颈，早期研究尝试了“零样本学习（Zero-Shot Learning, ZSL）”。 ZSL 的基本思路是：

  • 训练阶段只使用“已见类（seen classes）”的标注；
  • 测试时要求模型识别“未见类（unseen classes）”；
  • 借助语义词嵌入（word embeddings）或属性向量（attribute vectors）连接已见类与未见类的语义空间。

然而，ZSL 的局限性非常明显：

  1. 语义依赖过强：模型只通过词嵌入来推理未知类，无法充分利用视觉特征；
  2. 训练分布偏差：未见类在训练阶段被当作“背景”，导致模型忽略了新类别的潜在视觉特征；
  3. 泛化能力有限：ZSL 往往只能在语义空间中找到粗略的相似类，而难以真正识别复杂新目标。

三、开放词汇学习的提出与核心思想

针对 ZSL 的局限，作者引出了本文的主角——开放词汇学习（Open Vocabulary Learning, OVL）。 它的目标是：

让模型在只学习部分类别（base classes）的情况下，也能识别新类别（novel classes），并且能够灵活扩展到任意自然语言词汇表。

实现这一目标的关键在于——视觉-语言对齐（Vision-Language Alignment）。 开放词汇学习通过大规模的**视觉语言模型（Vision-Language Models, VLMs）**实现这种对齐，例如：

  • CLIP（Radford et al., 2021）
  • ALIGN（Jia et al., 2021）

这些模型通过海量图文对（image–text pairs）预训练，使图像特征与语言特征映射到同一语义空间中。这样，模型可以“理解”文本标签的含义，而不必在训练集中见过该类图像。

这一步革新，直接解决了传统 ZSL 的痛点：

  • 模型不再被限制于封闭标签集；
  • 语义空间的词汇量几乎无限大；
  • 可以利用海量图文弱监督数据（例如网络图片及其标题），显著降低标注成本。

因此，OVL 能让检测器/分割器在测试时“自动识别未训练类别”，成为现实世界可扩展视觉模型的重要方向。

四、语言数据的引入：从封闭集到开放语义

作者进一步指出，开放词汇学习之所以强大，还因为它把“语言数据”作为一种廉价、广覆盖的弱监督信号：

  1. 成本低：文字标签、标题等比标框或掩码的成本低得多；
  2. 语义广：语言数据中包含的词汇远超训练集标签；
  3. 语义细粒度：描述中往往包含对象的属性、动作和关系等信息；
  4. 可泛化性强：自然语言描述能覆盖许多长尾或未标注类别。

这意味着，模型可以借助语言监督来“自我扩展词汇表”，不再受限于人工标签，从而具备开放世界感知能力。

五、文献空白与本文贡献

在介绍完背景后，作者强调了本综述的必要性：

尽管近年来出现了许多与开放词汇相关的研究，但目前尚无系统综述梳理整个领域的脉络、方法、数据和基准。

因此，本论文的贡献在于三个方面：

  1. 全面系统化：首次系统总结开放词汇学习在检测、分割、视频理解、3D 场景理解中的最新进展；

2 BACKGROUND

本节首先阐述开放词汇及其相关概念的比较与概念定义,随后对开放词汇学习进行历史回顾并指出若干代表性工作。

接着介绍标准数据集与评估指标,同时提出适用于开放词汇目标检测和分割任务的统一符号体系。最后梳理相关研究领域。

2.1 Concept Definition

这一节是整篇综述的“理论地基”，它系统地阐述了 开放词汇学习（Open Vocabulary Learning, OVL）* 的形式化定义、数据假设、与相邻概念（Open-Set、Open-World、Zero-Shot、OOD）之间的数学区别。

一、问题起点：封闭集假设的局限

作者首先从最熟悉的场景——分类任务（Classification Task） 出发来讲解：

在传统监督学习（Supervised Learning）中，训练数据和测试数据被假设来自相同的封闭标签空间（Closed-set Label Space）。 也就是说，模型训练和推理时面对的类别集合完全一致。

数学上表示为：

Ctrain=Ctest

这种设定在早期的 ImageNet、COCO、ADE20K 等基准中很常见。

然而，这样的假设在现实中几乎不成立。 ——真实世界的类别是开放的（open-ended）： 新的物体、新的语义、新的场景不断出现。

因此，如果我们仍让模型在一个封闭类集合上学习，它会在遇到未见类别时完全失效。 这就是为什么研究者提出了一系列更开放的学习范式（如 Open-Set、Zero-Shot、Open Vocabulary），来缓解这个局限。

二、基本符号系统与任务设置

作者接着引入了统一的符号定义，这套符号贯穿全文：

符号 含义
CB Base classes（基类、训练集中有标注的类别）
CN Novel classes（新类、测试时才出现）
CL Language vocabulary（语言词汇表，可能包含远超训练集的语义词汇）
x 输入样本（图像或像素等）
y 类别标签（属于 CB 或 CN ）

于是，训练集与测试集可以形式化表示为：

  • 训练集：

    Dtrain={(xi,yi)},yi∈CB

  • 测试集：

    Dtest={(xi′,yi′)},yi′∈(CB∪CN)

也就是说，训练阶段只见到基类（Base），但测试时模型需要在更大的类集合（Base ∪ Novel）上做预测。

三、开放词汇学习的核心思想

在传统 ZSL 或 Open-Set 学习中，模型虽然也会遇到新类，但没有语言辅助；而在 开放词汇学习（OVL） 中，我们允许模型借助 语言相关数据（Language-related data） 来辅助学习。

例如：

  • 图像标题（captions）
  • 图文对（image-text pairs）
  • VLM（如 CLIP）的文本嵌入

这些“语言数据”作为辅助监督信号（auxiliary supervision），极大扩展了模型的语义覆盖范围。

作者特别强调两点动机：

  1. 低成本（Cost Efficiency） 语言数据不需要画框或标掩码，采集便宜、量大。 例如“一个穿蓝衣服的人在骑车”这样的标题，天然包含了语义信息。

  2. 词汇广（Vocabulary Extensibility） 标注集的 80 个类无法覆盖真实世界的语义，但自然语言却能表达几乎任意类别、属性或动作。 → 这使得模型的输出空间从“固定标签集”扩展到“开放语言空间”。

四、VLM 的关键作用：视觉与语言空间的对齐

在开放词汇学习中，视觉语言预训练模型（Vision-Language Models, VLMs） 是核心基石。

代表性模型：

  • CLIP (Radford et al., ICML 2021)
  • ALIGN (Jia et al., ICML 2021)

这些模型通过海量图文对 (image,caption) 的对比学习，将图像和文本映射到统一的语义嵌入空间。 换言之，它们学到了一个函数：

fimg(x)≈ftext(t)

只要图像 x 与文字描述 t 语义相符，它们的特征向量在空间中就会接近。

这就让检测/分割模型可以：

  • 把每个候选区域的视觉嵌入与一个庞大的文本词汇嵌入集合计算相似度；
  • 直接选择最相似的文本类别作为输出。

从而实现：

“只要我知道这个词的语义，就能识别这种物体”

这便是开放词汇学习的精髓。

五、形式化比较：与其他相似概念的区别

作者用一个非常关键的图（Fig. 1）总结了几种容易混淆的概念。 我们来逐个讲解：

概念 目标 测试阶段类别 模型行为
Open-Set Learning 区分已知和未知 CB∪{u} 只需识别未知并标记为“unknown”
Open-World Learning 动态地增量学习新类 随时间增加 CBt 可逐步扩展标签集
OOD Detection 检测分布外样本 不固定 只判断“是否在训练分布中”
Zero-Shot Learning (ZSL) 识别训练时未见过的类别 CN 借助词嵌入识别新类（但无图像级语言辅助）
Open Vocabulary Learning (OVL) 同时识别 seen & unseen 类 CB∪CN 借助语言模型扩展语义空间

换句话说：

  • ZSL 是 OV 的“特例”：OVL 允许利用更多弱标语言数据；
  • Open-Set/World 只需判断“未知”，但不必识别是哪一类；
  • OVL 则能具体命名新类（如“hedgehog”、“thermos bottle”），这是它最显著的优势。

六、开放词汇学习的正式定义

作者给出了 OVL 的正式数据定义：

  • 训练集：

    Dtrain={(xi,yi,li)},yi∈CB, li 是语言相关信息

    其中 li 可以是图像标题、语言描述、或来自 VLM 的文本嵌入。

  • 测试集：

    Dtest={(xi′,yi′)},yi′∈(CB∪CN)

目标是学习一个模型 f(x) ，能在无新增标注的情况下识别 CN 中的类别。

七、几个关键概念的对比

维度 Zero-Shot Learning (ZSL) Open-Set Recognition (OSR) Open Vocabulary Learning (OVL)
训练数据 仅含“已见类” (Seen Classes) + 对应的语义信息 (如属性、词向量) 仅含“已知类” (Known Classes) 含“已见类” + 大量图文对数据 (Image-Text Pairs / Captions)
核心任务 将样本分类到预先定义好的 “未见类” (Unseen Classes) 集合中 正确分类“已知类”，并能拒绝 所有不属于已知类的“未知类” 识别或检测由任意开放文本 (Arbitrary Text) 描述的新类
新类定义 新类列表在测试时必须提供 (例如：["斑马", "长颈鹿"]) 无需定义新类 (所有新类都被视为一个 "Unknown" 类别) 无需预定义 (通过任意文本查询 prompt 来定义)
关键技术依赖 ✅ 是 (语义对齐) 依赖“语义信息”（如词向量、VLM） 建立 视觉-语义 映射 ❌ 否 (通常) 依赖“分布外检测” (OOD) 学习已知类的边界 ✅ 是 (大规模VLM) 依赖 CLIP, ALIGN 等模型的 开放世界视觉-语言对齐能力
核心挑战 域偏移 (Domain Shift) (从“已见类”到“未见类”的泛化) 开放空间风险 (如何高效识别高度多样化的未知) 细粒度理解与定位 (如何处理复杂的、组合性的文本描述)
主要应用 学术任务, 大规模物种分类 异常检测, 安全关键系统 (如自动驾驶) 开放词汇检测/分割, 图像检索, 交互式AI

(a) 开集学习 (Open-Set Learning / OSR)

  • 训练 ( Dtrain )：仅使用 CB 的数据进行训练 。
  • 评估 ( Deval )：测试集包含 CB 和“未知”类别。
  • 目标：在评估时，模型需要能正确分类 CB 的样本，并拒绝所有不属于 CB 的样本，将它们统一标记为“未知” ( u )。
  • 与 OVL 的核心区别：OSR 的目标是**“识别已知，拒绝未知”。它不需要**（也不能）进一步区分未知样本具体属于哪个新类别（例如，它只知道“这不是狗”，但不知道“这是一只猫”） 。而 OVL 的目标是**“识别已知，也识别未知”** 。
  • 图1(a) 的诠释：新类别（不同形状）都被归类为“未分类” (not classified) [cite: 96]。

(b) 零样本学习 (Zero-Shot Learning / ZSL)

  • 训练 ( Dtrain )：仅使用 CB （在 ZSL 语境下常称为“可见类”，seen classes）的数据进行训练。
  • 评估 ( Deval )：测试集仅包含 CN （“未见类”，unseen classes）的样本 。
  • 目标：模型必须在 CN 类别中做出明确的分类 。
  • 与 OVL 的核心区别：
    1. 目标相似，设定不同：ZSL 的目标也是识别 CN 。但 ZSL 在训练时无法访问 CL 这样的大规模辅助语言数据 。ZSL 依赖的是 CB 和 CN 之间共享的、预先定义好的语义信息（如词嵌入或属性向量）。
    2. OVL 更实用：OVL 通过在训练中引入 CL （如图文标题），极大地增强了模型对新类别的泛化能力，解决了 ZSL 将新物体视为“背景”而导致效果不佳的问题
  • 图1(b) 的诠释：模型必须分类 CN ，但其训练环境中没有 CL （那个大的淡蓝色圆圈）的辅助

(c) 开放词汇学习 (Open Vocabulary Learning / OVL)

  • 图1(c) 的诠释：OVL 的目标与 ZSL 相同（分类 CN ），但它的关键区别在于，它在训练时就可以利用 CL （大的淡蓝色圆圈）的知识

(d) 开放世界学习 (Open World Learning / OWL)

  • 目标：这是一个动态的、持续的过程
  • 流程：
    1. 像 OSR 一样，识别已知 CB ，检测未知 u
    2. 关键步骤：将检测到的未知 u 交由人类标注
    3. 模型增量式地学习 (incrementally learning) 这些新标注的类别，使其成为新的“已知”类别
  • 与 OVL 的核心区别：OVL 是一个静态的评测设置（训练一次，测试一次）。OWL 是一个动态的系统，它包含了 OVL（或 OSR）的能力作为其“发现新事物”的第一步，并增加了“增量学习”的闭环

(e) 分布外检测 (Out-of-Distribution / OOD)

  • 目标：OOD 关注的是数据分布的差异
  • 设定：训练数据来自 Ptrain 。测试时，需要识别出那些来自不同分布 POOD 的样本 xi′
  • 与 OVL 的核心区别：OOD 是一个更宽泛的概念。一个 OOD 样本可能是一个 CN 类别（OVL 关注的），但也可能是一个 CB 类别但在不同光照、风格或损坏下的图像。OVL 专注于语义类别的新颖性，而 OOD 专注于数据分布的新颖性

2.2 History and Roadmap

OVL 的发展脉络

1. 早期探索与概念起源

  • 面临的挑战：研究人员一直面临一个棘手的问题，即如何在真实世界中定位和分类任意物体，因为现有数据集的类别（词汇表）非常有限 。
  • 概念雏形 (2017)：论文指出，“开放词汇”这一概念最早在场景理解（scene understanding）领域被提出 。2017 年的开创性工作 [41] 尝试构建一个联合图像像素和词汇概念的嵌入框架 。
  • NLP 的推动 (2019)：随着 NLP 领域 BERT [9] 模型的兴起，多模态预训练（即结合视觉和语言）开始受到广泛研究 。

2. 关键节点：OVL 范式的正式提出 (2021)

2021 年是 OVL 发展史上至关重要的一年，两大关键进展在这一年发生：

  1. OVOD 概念的提出 (OVR-CNN)：
    • 受视觉语言预训练的启发，OVR-CNN [11] 首次正式提出了**“开放词汇目标检测” (Open Vocabulary Object Detection)** 的概念 。
    • 它的核心贡献是利用标题数据 (caption data) 来连接新类别（novel classes）的语义和视觉区域，从而使模型能够检测这些在训练时未见过（但可能在标题中出现过）的物体 。
  2. 催化剂：CLIP 的发布：
    • 同年，CLIP [15] 发布并开源 。
    • CLIP 是一个大规模视觉语言模型 (VLM)，它在海量的图文对上进行了预训练，实现了强大的图像和文本特征对齐能力。它的出现，为 OVL 提供了一个极其强大的基础工具（或“知识库”）。

3. OVL 的爆发期：VLM 知识的应用 (2022至今)

CLIP 的出现迅速催生了一批将 VLM 知识应用到下游密集型任务（如检测、分割）的工作：

  • ViLD (2022)：这是第一个利用 CLIP 知识来构建开放词汇目标检测器的工作 。它的核心思想是（如我们上一节课所学）将 CLIP 的知识“蒸馏”到检测器中。

  • LSeg (2022)：几乎在同一时间，LSeg [14] 率先探索了如何利用 CLIP 的知识来完成语言驱动的语义分割任务 。

    在这些开创性工作之后，OVL 领域进入了**“爆发期”** 。

4. 近期趋势：更强基础模型的融入

  • SAM (2023)：Segment Anything Model (SAM) [42] 的提出，构建了一个在十亿级掩码上训练的分割基础模型 。通过与 CLIP 结合，SAM 也能实现强大的零样本分割能力 。
  • LLM (大语言模型)：近期，随着大型语言模型 (LLM) [43] 的飞速发展，研究人员开始探索如何将 LLM 中蕴含的更丰富的语言知识和常识推理能力嵌入到 OVL 的多模态架构中，这使 OVL 成为一个更具前景的方向 。

2.3 Tasks, Datasets, and Metrics

本节内容的作用是为 OVL 研究“划定战场”。它明确了 OVL 主要在哪些任务上进行研究，使用哪些公开的数据集作为“靶场”来验证模型性能，以及使用哪些统一的指标来“打分”。

1. 任务 (Tasks)

本节定义了 OVL 的“研究对象”是什么。

  • 核心目标：论文首先明确，所有 OVL 任务的中心目标是相似的，即**“借助大型词汇库的知识来识别新类别”** 。
  • 任务范围：OVL 已经涵盖了计算机视觉中的多种任务 。
  • 本综述的重点：这篇论文声明，它将主要关注场景理解 (Scene Understanding) 相关的任务，具体包括：
    • 目标检测 (Object Detection)
    • 实例分割 (Instance Segmentation)
    • 语义分割 (Semantic Segmentation)
    • 目标跟踪 (Object Tracking)
  • 其他相关任务：除了上述重点，论文也提到了 OVL 还在以下任务中被研究：
    • 视频理解 (Video Understanding)
    • 3D 场景理解 (3D Scene Understanding)
    • 开放词汇属性预测 (Open Vocabulary Attribution Prediction)
    • 视频分类 (Video Classification)
    • 点云分类 (Point Cloud Classification)

2. 数据集 (Datasets)

本节列出了用于评估上述 OVL 任务的主流公共数据集。

  • 对于目标检测 (Object Detection)：

    • COCO [5]：这是一个非常通用和基础的数据集 。

    • LVIS [49]：这是一个大规模词汇的数据集，专为长尾识别（即许多类别样本稀少）而设计，非常契合 OVL 的设定 。

    • v3Det [50]：被认为是一个更具挑战性的数据集，因为它包含超过 10,000 个类别 。

  • 对于图像分割 (Image Segmentation)：

    • COCO [5]
    • ADE20k [51]：一个用于场景解析（semantic scene parsing）的数据集 。
    • PASCAL-VOC 2012 [52]
    • PASCAL-Context [53]
    • Cityscapes [54]：专注于城市街道场景的数据集 。

  • 对于视频分割和跟踪 (Video Segmentation and Tracking)：

    • VSPW [55]
    • Youtube-VIS [56]
    • LV-VIS [57]
    • MOSE [58]
    • TAO [59]

3. 评估指标 (Metrics)

本节定义了如何“量化”OVL 模型的性能，即“评分标准”。

  • 对于检测任务 (Detection Tasks)：

    • mAP (mean Average Precision)：即平均精度均值，是目标检测的标准指标 。

    • mAR (mean Average Recall)：即平均召回率均值（注：原文 误写作 mAP，但上下文意指 mAR）。

    • OVL 特点：在 OVL 中，研究者会分别计算基础类别 (APbase) 和新类别 (APnovel) 的 mAP 。

  • 对于分割任务 (Segmentation Tasks)：

    • 语义分割 (Semantic Segmentation)：使用 mIoU (mean Intersection over Union)，即平均交并比 。
    • 实例分割 (Instance Segmentation)：使用基于掩码（Mask）的 mAP 。
    • 全景分割 (Panoptic Segmentation)：使用 PQ (Panoptic Quality)，即全景质量 。

总结

第 2.3 节为 OVL 设定了清晰的研究框架。它告诉我们：

  1. “做什么？”（任务）：主要是检测和分割，并向视频和 3D 扩展。
  2. “在哪里做？”（数据集）：COCO, LVIS, ADE20k 是最核心的几个基准。
  3. “如何评价？”（指标）：mAP, mIoU, PQ 是金标准，并且必须分开考察模型在“基础类别”和“新类别”上的表现。

2.4 Related Research Domains

在 2.1 节中，我们已经从“定义”上区分了 OVL 和其他几个概念。2.4 节则是从“方法论”和“研究目标”上，对这些相关领域进行了更深入的展开和回顾。这有助于我们更清晰地理解 OVL 技术的“生态位”，即它解决了哪些其他领域没有解决的问题，又借鉴了哪些领域的技术。

本节内容可以看作是对 2.1 节概念对比的详细“文献综述”补充。

1. 开集识别 (Open-Set Recognition, OSR)

  • 目标：OSR 的核心挑战是在分类任务中识别“未知”类别 。在测试时，它必须能够正确分类在训练中见过的“已知”类别，并拒绝那些未见过的“未知”类别 。
  • 方法分类：
    • 判别式模型 (Discriminative models)：这类方法的核心是“划清界限”，它们专注于增强“已知”类别和“未知”类别之间的边界或裕度（margin）。
    • 生成式模型 (Generative models)：这类方法通过生成新的实例或特征 ，来提升系统识别新类别的能力 。
  • 与 OVL 的关系：OSR 的终点是“拒绝”，即标记为“未知”。而 OVL 的目标是“识别”，即不仅知道它是未知的，还要知道它具体是“哪种”新类别。

2.开放世界学习 (Open World Learning)

  • 目标：这是一个更进一步、更动态的设定。它要求模型不仅能识别和标记新的“未知”类别 ，还必须能够在这些新类别被标注后，增量式地学习 (incrementally learning) 它们，并将它们纳入自己的知识体系，而无需从头重新训练 。
  • 方法：[38] 首次提出了开放世界识别。近期的工作也开始探索开放世界的目标检测 。
  • 与 OVL 的关系：OVL 通常是一个“静态”的任务（训练一次，测试一次）。而 OWL 是一个“动态”的系统，它要求模型能持续适应新知识。OWL 可以看作是 OSR（发现未知）和增量学习（学习新知）的结合体。

3. 分布外检测 (Out-of-Distribution, OOD)

  • 目标：OOD 专注于检测那些与训练数据“不同”的数据 。
  • 方法分类：
    • 基于分类的方法：例如基于模型输出的置信度 。
    • 基于密度的方法：通过对数据密度建模来检测 OOD 样本 。
    • 基于距离的方法：在特征空间中使用距离度量来识别 OOD 实例 。
    • 基于重构的方法：利用模型的重构能力来检测 OOD 。
  • 与 OVL 的关系：OOD 是一个比 OSR 更宽泛的概念。OOD 样本不仅可以是“新类别”，也可能是“已知类别”但在不同环境下的样本（如损坏、风格变化）。OVL 更具体，它专注于“语义上”的新类别。

4. 零样本检测与分割 (Zero-Shot Detection and Segmentation)

  • 目标：检测或分割那些在训练期间未曾遇到过的类别 。

  • 方法分类：

    • 判别式方法 (Discriminative methods)：代表作 SPNet 15。它将每个像素映射到一个语义词嵌入空间，然后使用一个固定的语义词嵌入矩阵来预测类别概率 。
    • 生成式方法 (Generative methods)：代表作 ZS3Net 。它首先训练一个生成模型，根据词嵌入来合成“未见类”的像素级特征 。然后，利用这些合成特征，以监督学习的方式训练分割模型 。

  • 局限性：论文指出，这种将 ZSL 视为“像素级零样本分类”的范式不够鲁棒，因为文本嵌入（如“汽车”）通常是用来描述整个物体或区域的，而不是单个像素的 。

  • 与 OVL 的关系：这是与 OVL 关系最密切的领域。OVL 中的许多方法都基于 ZSL 的设定（例如用语言嵌入替换固定分类器）。但 OVL 解决了 ZSL 的核心痛点：传统 ZSL 缺乏新类别的知识，泛化能力有限 ，而 OVL 明确地利用（VLM 带来的）大规模语言知识来弥补这一缺陷，使其更强大、更实用。

5. 长尾检测与分割 (Long-tail Object Detection and Segmentation)

  • 目标：解决数据集中“类别不均衡”的挑战 。即模型如何处理那些在训练集中出现次数极少（“尾部”）的类别。
  • 方法分类：
    • 数据重采样 (Data re-sampling)：例如，在图像级别 24或更精细的实例/特征级别 25 增加“尾部”类别样本的采样频率。
    • 损失重加权 (Loss re-weighting)：在训练期间，重新平衡正负样本的损失权重 26。
    • 解耦训练 (Decoupled training)：例如，将训练过程分为表示学习和分类器校准两个阶段 27。
  • 与 OVL 的关系：长尾学习的目标是提升稀有（Rare）类别（即训练集中样本量 K > 0 且 K 很小）的性能 。它无法应用于新（Novel）类别（即 K = 0）的检测 。

6. 少样本检测与分割 (Few-Shot Detection and Segmentation)

  • 目标：仅使用**极少数（例如 K=1 或 K=5）**的标注样本，来扩展模型对新类别的检测/分割能力 30。

  • 方法分类：

    • 检测 (FSD)：例如，TFA 31引入了一个简单的两阶段微调方法；DeFRCN 32 则解耦了 RPN 特征和 RoI 分类的训练。

    • 分割 (FSS)：

      • FSSS (语义分割)：通常是从“支持集”图像中构建类别原型 (prototypes)，然后通过计算“查询集”图像特征与原型之间的相似度来进行分割 。

      • FSIS (实例分割)：例如，双分支方法 引入一个额外的“支持分支”来计算类别原型或重加权向量，以辅助分割器识别目标。

  • 与 OVL 的关系：这是最关键的区别之一。少样本学习（Few-Shot）仍然需要在训练（或微调）阶段提供 K 个（K>0）新类别的标注样本 。而 OVL 的要求更严格，它要求在 K=0（即完全没有新类别的标注图像）的情况下进行识别。

3 METHODS: A SURVEY

3.1 Preliminary

作者先把“传统的目标检测 / 分割模型长什么样”讲清楚，好让后面解释“怎么把它们改造成 open-vocabulary 模型”。

他们主要分成两大类：

  1. Pixel-based（像素为中心）检测与分割
  2. Query-based（查询为中心）检测与分割

下面我按 paper 的逻辑，一块块拆给你。

一、Pixel-based Object Detection and Segmentation

作者先说：场景理解任务可以分成两种层级：

  • 语义级（semantic-level）任务：只在乎“这个像素是什么类”，不区分同类的不同实例
  • 实例级（instance-level）任务：不仅要知道“是什么类”，还要区分同类的不同物体

1. 语义分割：把每一个像素分类

他们以 semantic segmentation（语义分割） 为例。

“It was typically approached as a dense pixel classification problem, as initially proposed by FCN [140].”

FCN（Fully Convolutional Network）框架的核心想法：

  1. 用 CNN 提取整张图的特征（downsample，一路卷积+池化）
  2. 再通过上采样/反卷积，把特征图恢复到接近原图大小
  3. 在每个像素位置上，接一个分类头，对像素做多类分类

之后的工作基本都在 FCN 框架上做各种增强。作者把这些增强分成几个方向：

(1) 更好的 encoder–decoder 结构

  • Encoder 负责抽象高级语义（越来越小的 feature map，通道数越来越多）
  • Decoder 负责逐步恢复空间细节（上采样 + skip connection 等）

改进点包括：

  • 如何更好地把浅层的细节信息（边缘、纹理）和深层的语义信息（类别）结合
  • 如何在恢复分辨率时减少“糊边”、伪影

(2) 大卷积核 / 更大感受野

  • 使用 大卷积核 或者 空洞卷积（dilated conv） 让每个像素的预测看到更大的上下文区域
  • 对于分割来说，一个像素要看足够大的区域，才能知道自己到底是“车顶”还是“路面反光”，这就是大感受野的意义

(3) 多尺度 pooling & 多尺度特征融合

  • 现实中的目标尺寸差别很大：有近处的大车、远处的小人
  • Multi-scale pooling：在不同尺度上做 pooling（例如 PSPNet 那种 pyramid pooling）
  • Multi-scale feature fusion：融合高分辨率 + 低分辨率的特征（例如 FPN 样式的 top-down 融合）

直觉：

“我既想看整体场景（大尺度），又要看边缘细节（小尺度），再综合判断每个像素。”

(4) Non-local / 自注意力建模

  • Non-local / self-attention 让远处像素也能直接互相“看见”
  • 例如，画面左边一半的“马”可以帮助右边半截“马”判断自己也是“马的一部分”

(5) 更好的边界建模

  • 传统 FCN 容易把物体边缘弄得模糊

  • 一些工作专门设计 boundary-aware 模块：

    • 单独预测边界，再和主体 mask 融合
    • 或在 loss 中对边界区域加权，让网络更重视边缘

(6) Transformer 之后：自注意力替代卷积 Head

“After the transformer was proposed … several works propose variants of self-attention operators to replace the CNN prediction heads.”

  • 原来最后一层是卷积 + softmax

  • 现在很多方法直接用 Transformer / self-attention 作为预测头：

    • 更好地利用全局上下文
    • 对复杂场景（多物体、遮挡）更鲁棒

2. 实例级：目标检测 & 实例分割

接着作者说实例级任务，以 object detection + instance segmentation 举例。

2.1 目标检测（Object Detection）

目标：

“detect each instance box and classify each instance”——找到每个物体的边界框，并预测类别。

传统上分为两大类：

(1) Two-stage 方法

代表：Faster R-CNN 系列。

  • 第一阶段：Region Proposal Network (RPN)

    • 在 feature map 上打很多 “anchor”（先验框，大小/长宽比不同）
    • 预测每个 anchor 是前景还是背景 + 粗略回归框
    • 挑选 top-K 个 Proposal，称作 RoI（Region of Interest）

  • 第二阶段：RoI Head

    • 把每个 RoI 特征裁剪出来（RoIAlign）
    • 对每个 RoI 做分类 + 边界框回归精修

特点：

  • 显式地“先找可能是物体的区域，再精细分类回归”
  • 召回率更高、对小目标更友好，所以作者说： “Two-stage approaches are more commonly used in open vocabulary object detection tasks.”

(2) One-stage 方法

代表：RetinaNet、YOLO等。

  • 不再有单独的 RPN，直接在密集的 anchor/特征点上做分类+回归

  • 优点：

    • 设计更简单、速度更快
    • 加上 Focal Loss、FPN 之后，精度可以追上甚至超过 two-stage 方法

(3) Anchor 的概念

“The anchors are the pixel locations where the objects are possibly emerging.”

  • 可以理解成：在特征图上的每个位置，预先假设一些可能存在物体的框
  • 网络训练的目标：调节这些框的位置与大小 + 判断里面是什么类

2.2 实例分割（Instance Segmentation）

实例分割比检测更难： 不仅要把框框出来，还要得到“每个物体的精确像素区域 mask”。

作者分成两大类方法：

(1) Top-down 方法

流程：

先检测框 → 对每个框预测 mask

  • 在检测器上加一个 mask head（例如 Mask R-CNN）

  • 每个 RoI 特征单独送进 mask head，输出这一实例的像素级 mask

  • 改进方向：

    • 不同的 FCN mask head
    • 不同的 mask 编码方式（多通道 mask、shape basis 等）
    • 动态卷积核（根据实例特征生成卷积核）

优点：

  • 与检测算法高度兼容，直观好用

缺点：

  • 对检测框质量高度依赖（框偏了，mask 也会错）

(2) Bottom-up 方法

流程：

先做语义分割，再把同类不同实例“聚类分开”

  • 首先对每个像素预测语义类别（和 semantic segmentation 类似）
  • 然后根据边缘、中心点、embedding 等特征，对同类像素进行聚类，划成不同实例

优点：

  • 不依赖框，理论上对密集场景更友好

缺点：

  • 聚类过程比较 tricky，容易粘连或过分裂

二、Query-based Object Detection and Segmentation

在 pixel-based 方法之后，作者介绍了 Transformer + Query-based 的范式，以 DETR 为代表。

这类方法的核心思想是：不要在空间上密集地扫 anchor，而是让若干“object queries”自己去“找”物体。

1. DETR 的基本结构

“One representative work is Detection Transformer (DETR). It contains a CNN backbone, a standard transformer encoder, and a standard transformer decoder.”

  • CNN Backbone：提取图像特征（类似 ResNet）

  • Transformer Encoder：把特征图 flatten 后做自注意力，捕捉全局关系

  • Transformer Decoder + Object Queries：

    • 使用 N 个 learnable 的 query 向量
    • 每个 query 在 decoder 里和图像特征交互，学习去表示“一个可能的物体”

2. 一对一匹配（Bipartite Matching）

训练时，DETR 不再用 NMS，而是设计了**“预测集合”和“真值集合”之间的一对一匹配问题**：

  • 对每个 predicted object（来自一个 query），有一个预测的：

    • 类别
    • 边界框（或 mask）

  • 用 Hungarian Algorithm 求解“预测集合 ↔ 真值集合”的最优匹配，使得整体损失最小

  • 匹配后，对每个 matched pair 用分类损失 + box 回归损失训练；没匹配到真值的 query 被视为“no object”

好处：

  • 不用手工设计 anchor
  • 不用 NMS 去除重复检测
  • 一个 query 对应一个实例，自然就是“集合到集合”的预测

3. 用 query 做分割：Mask Transformer 系列

当 DETR 被扩展到实例分割时：

  • 每个 object query 不仅预测类别和框，还预测一个 mask
  • 常见做法：把 object query 当作一个向量，与 decoder 的特征图做内积，得到一张 mask（本质上就是 query 作为“模板”去激活相应区域）

后续一系列 mask transformer 工作进一步强化了这一范式：

  • Max-Deeplab：完全去掉 box head，只保留 query + mask 的路径，成为纯 mask-based 的分割模型

  • K-Net：用“mask pooling”把 mask 区域 pooling 成一个向量，然后用动态卷积更新对应的 query

  • MaskFormer / Mask2Former：

    • 发现：“语义、实例、panoptic 三种分割任务，其实都可以看成**‘预测若干 mask + 给每个 mask 一个类别’**，不必区分不同任务”
    • Mask2Former 进一步使用 masked cross-attention，强制每个 query 只关注自己对应的区域

作者总结：

“Query-based approaches are stronger and simpler. They cannot directly detect novel classes but are widely used as base detectors and segmenters in open vocabulary settings.”

也就是说：

  • 这些 query-based 模型本身还是 closed-set 的（只能识别训练过的类别）
  • 但因为结构统一、表现强，非常适合作为 open-vocabulary 方法的“骨架”：之后只要把分类头换成 VLM 的文本 embedding，就能让它识别没见过的新类——这就是后面 3.2、3.3 要讲的东西。

三、小结

3.1 做了三件事：

  1. 回顾传统的 pixel-based 语义/实例分割和检测

    • FCN 系列：密集像素分类
    • RPN + RoI 的两阶段检测，和 one-stage 检测
    • Top-down / bottom-up 实例分割

  2. 介绍新一代 query-based（DETR, MaskFormer 系列）框架

    • Backbone + Transformer + object queries
    • Hungarian matching 实现集合到集合的预测
    • 用 query → mask 的方式统一各种分割任务

  3. 为后面 open-vocabulary 架构打基础

    • 这些 close-set 检测/分割器，是后面所有 open-vocabulary 方法的“底座”

    • 后续章节会在这个底座上：

      • 加进 VLM（CLIP、ALIGN 等）的文本 embedding
      • 或用 caption / image-text pair 做预训练
      • 让模型能识别训练集中从未标注过的新类别

3.2 Vision Language Modeling

这一节主要讲解如何通过视觉语言模型（Vision-Language Models, VLMs）将视觉信息和文本信息结合起来，以实现open-vocabulary 学习。在这一部分，作者主要介绍了两个关键技术：视觉语言模型的训练和它们如何应用于目标检测、分割等任务中。

我将详细拆解这部分内容，并且以更直白的语言帮助你理解。我们可以从以下几个方面来讨论：

1. 视觉语言模型的预训练（Vision-Language Pre-training）

（a）VLMs的预训练基础

视觉语言模型（VLM）通过大规模的图像-文本配对来预训练，以便学习图像和文本之间的关系。这个过程使得模型能够理解图像内容与文本描述之间的关联。

关键概念：

  • 训练目标：VLM 的目标是通过输入图像和对应的文本描述，让模型学会如何对齐视觉和语言信息。这通常是通过简单的任务（如：匹配图像和描述）来实现的。

例如：给定一张猫的图片，文本描述为“a cat sitting on a sofa”，VLM 需要学会将图像特征和描述中的“cat”这个词对齐起来。

VLM 的训练目标很简单：通过图像和文本的对比，模型学会将图像和文本映射到同一特征空间。这意味着它能够在不同的输入（图像或文本）之间找到关联和相似性。

（b）如何训练 VLMs

视觉语言模型的训练通常采用以下两种方法之一：

  1. 双流网络（Two-Stream Networks）：

    • 这类方法使用两种网络来分别处理图像和文本。图像通过视觉网络（如卷积神经网络或视觉变压器）提取特征，而文本则通过文本网络（如 BERT 或 Transformer）提取语义信息。
    • 训练过程中，两个网络的输出通过一个融合层进行对齐，最终模型学会图像和文本之间的关联。

  2. 单流网络（Single-Stream Networks）：

    • 这类方法将图像和文本融合到同一个神经网络中，模型在学习过程中直接对图像和文本进行共同处理。

代表性模型：

  • CLIP（Contrastive Language-Image Pre-training）：CLIP 是一种流行的 VLM，通过大量的图像-文本配对进行预训练，使得图像和文本可以在同一特征空间中进行比较。CLIP 可以用于广泛的任务，包括目标检测、图像搜索等。
  • ALIGN（A Large-Scale ImaGe-Text Embedding）：与 CLIP 类似，ALIGN 也是一种基于对比学习的视觉语言模型，通过大规模的图像和文本数据进行预训练，能够更好地理解视觉和语言之间的关联。

2. 如何将 VLM 应用到 open-vocabulary 设置中

（a）Open-Vocabulary 学习的核心思想

VLM 的核心优势之一是能够在没有明确训练样本的情况下，识别新的（未见过的）类别。open-vocabulary 的学习正是基于此，目的是扩展传统模型的能力，让其在面对新类时，不需要额外的标注数据。

  • 传统模型的限制：传统的目标检测模型通常只能识别训练时见过的类别（即 close-set 模型）。例如，一个训练过的检测模型只能识别 COCO 数据集中包含的 80 个类别。
  • Open-Vocabulary 模型的优势：通过使用 VLM，如 CLIP，我们能够利用模型从大规模文本数据中获得的语言知识，来扩展模型的类别识别能力。

例如：

  • 假设我们训练了一个 open-vocabulary 模型，模型在训练时使用了 COCO 数据集的类别（如“猫”、“狗”等），并利用 CLIP 学习了文本和图像的关系。
  • 当模型遇到一张没有标注的新图像（比如某个新的动物，叫做“犰狳”），它能够基于图像和文本之间的相似性，推断出这张图像属于“犰狳”类别，即使在训练数据中没有这个类别。

（b）通过 VLM 实现 Open-Vocabulary 检测与分割

VLM 使得模型能够基于文本描述识别出未见过的类别。以目标检测为例：

  • 传统的目标检测方法通常需要为每个类别提供大量的标注数据（例如：每张图片的标注框）。但在 open-vocabulary 设置中，VLM 提供了一个新的方案：模型通过 语言描述 来扩展它的识别能力，无需为每个新类别提供单独的标注数据。

VLM 在 Open-Vocabulary 检测中的应用

  1. 文本描述辅助检测：
  • 通过将文本信息和图像结合，模型可以将图像中看到的物体与文本中的词汇（如“汽车”、“猫”、“瓶子”）对齐，进而识别未曾见过的物体。

  1. 结合 VLM 和现有检测器：

    • 一些方法将传统的目标检测框架（例如 Faster R-CNN、DETR）与 VLM 结合，通过 VLM 提供的文本 embedding 来指导检测过程。这样，检测器不仅能够检测训练时见过的物体，还能够识别新类别。

3. 视觉-语言模型的优势与挑战

优势

  • 零-shot 识别：通过对比学习，VLM 不仅能理解已见过的物体，还能通过语言对新物体进行识别。
  • 扩展性：VLM 能够扩展到开放词汇空间（open-vocabulary space），避免了传统模型的闭集限制。

挑战

  • 对齐问题：图像和文本的特征空间不完全相同，如何有效对齐它们仍然是一个挑战。
  • 计算资源需求：VLM 通常需要大规模的计算资源，尤其是在预训练阶段，训练 CLIP 这样的模型需要非常大的数据集和强大的计算能力。
  • 推理时的效率：尽管 VLM 在零-shot 任务中表现出色，但其推理过程可能较为缓慢，尤其是当模型需要对大量不同类别进行分类时。

4. 总结

3.2 节的核心内容是：通过 视觉语言模型（VLM），我们能够使目标检测、分割等任务从传统的 close-set 模型，扩展到 open-vocabulary 模型。这是通过将视觉信息与文本信息结合来实现的。通过 VLM，模型能够利用语言描述对未见过的类别进行识别，从而大大提高了模型的通用性和灵活性。

VLMs 的应用为 open-vocabulary 学习提供了强大的支持，尤其是在处理零-shot 任务时，能够利用训练时没有的类别进行推理，展现出强大的扩展性与应用潜力。

3.3 Open Vocabulary Object Detection

论文将 open-vocabulary object detection (OVD) 方法分为五大方向：

“knowledge distillation, region text pre-training, training with more balanced data, prompting modeling, and region text alignment”

这五类方法的目的都是解决一个核心需求：

让传统 close-set 检测器具备识别 novel 类别（训练集中没有的类别）的能力。

实现方式是：

  • 将 VLM（如 CLIP）中大规模、开放词汇的语义知识
  • “蒸馏” 或 “对齐” 到原本只能检测 base 类别的 detector 中。

📌 为什么需要特别的 OVD 方法？

如论文图 5 所示（页面上方图示），open-vocabulary 的思想是：

  • 用 VLM 提供的文本 embedding 替代 detector 的固定分类器权重
  • 对每个 proposal 计算它与所有类别文本 embedding 的相似度来预测类别

但问题是：

close-set detector 的视觉特征并没有与 VLM 的语言空间天然对齐！

因此 3.3 节介绍的五大类方法就是为了解决 “如何把 detector 的视觉 embedding 与 VLM 的文本 embedding 对齐” 这个关键问题。

① Knowledge Distillation（知识蒸馏）

论文开篇指出：

VLM 的知识远大于 close-set detector，因此最直接的想法就是蒸馏。

🔍 基本思想：

  1. 让 close-set detector 的 region feature 学习模仿 VLM 的 region-level 语义表示
  2. 让 detector 预测的类别分布更接近 VLM 的文本相似度分布

换句话说，就是把 “CLIP 的开放语义空间” 注入到检测器中。

🔧 典型方法（文中提及的代表作）

  • OVR-CNN
  • ViLD
  • PB-OVD

它们并未在这一段直接出现，但在论文其他地方明确归类为“知识蒸馏”方法（例如 Fig.4 场景图中，OVR-CNN、ViLD、PB-OVD 都位列 distillation 一行）。

🎯 方法常见结构：

(A) 蒸馏 region-level embedding

  • 使用 CLIP 的 image encoder 提取 region-level feature
  • 让 detector 的 RoI feature 去逼近这一表示 （例如：L2 loss / cosine similarity loss）

(B) 蒸馏文本对齐信息

  • 计算 detector 输出的分类 logits
  • 逼近 CLIP(text, region) 的相似度分布
  • 实现 novel 类别的 zero-shot 能力

🌈 优点

  • 利用 CLIP 的强大开放词汇知识
  • 不需要改 detector 的架构
  • 训练简单、泛化好

⚠️ 局限

  • 蒸馏不完全，只能“近似”对齐
  • 对长尾类别仍有限制（论文在 4.1 小节指出 base-class overfitting）

② Region-Text Pre-training（区域-文本预训练）

紧接在知识蒸馏方法之后，论文提出第二类：

Region-text pre-training（区域级别的视觉-语言预训练）

虽然文中对该类方法篇幅不多，但它属于重要类别。

🔍 核心思想：

不是先训练 detector 再蒸馏，而是直接训练一个 region-level VLM，使其天然理解 bounding box 与文本之间的对应关系。

与 CLIP 仅做 image-text 对齐不同，region-level pretraining 需要 “区域特征 ↔ 文本描述” 对齐。

🔧 实现方式可能包括（根据 OVD 标准做法）：

  1. 用 region caption 数据训练（如：Visual Genome 的 region caption）
  2. 将每个 RoI 与对应文本进行对比学习（contrastive learning）
  3. 得到一个 region-aware 的视觉-语言空间
  4. 检测器可直接使用该空间进行分类（dot product with text embedding）

在 Fig.4 的方法组织图中，诸如 MaskCLIP、OpenSeg、GLIP 一类模型与 region-text pretraining 有直接关系（图中对应 explicit caption data / region-text 训练路径）。

🌈 Region-text pretraining 的优势：

  • 比全局 image-text 更适合检测任务
  • 让模型天然理解局部物体的语义，与文本更强耦合
  • 更适合大型类别空间（例如 V3Det 的几千类）

⚠️ 局限：

  • 依赖大量 region-level caption 数据
  • 标注贵、规模有限（论文 10:0 指出 region caption 难以收集）

📌（本次总结）

方法类别 核心思路 优点 局限
Knowledge Distillation 把 CLIP 的语义空间蒸馏给 close-set detector 简单、有效、架构兼容性强 蒸馏不完全，仍包含 close-set 偏差
Region-Text Pre-training 在区域级别直接做视觉-语言预训练 对检测更自然、对齐效果好 region caption 数据有限，成本高

③ Training with More Balanced Data（使用更均衡的数据进行训练）

论文明确将其作为 OVD 的第三大方向：

“…training with more balanced data…”

🔍 背景问题：

在多数数据集（COCO、LVIS）中：

  • base 类（训练时出现的类别）数量远多于 novel 类
  • 除了类别数量不均衡，每类的样本数也严重不均衡（长尾问题）

这导致：

检测器在训练时对 base 类过拟合，对 novel 类的泛化大幅下降。

论文在“Challenge”部分指出：

“base classes overfitting” 是最核心问题之一

🔧 解决方案：

将 VLM 的对齐能力 + 更均衡的数据设计结合起来。

◼ 方法策略包括：

(1) 使用外部的长尾 / 大词汇数据集

如：V3Det（13k 类）

  • 可以大幅缓解 base 类的独占问题
  • 增强模型的语义覆盖范围

(2) 数据采样（re-sampling）或 reweighting

部分方法会：

  • 降低 frequent 类的采样权重
  • 增强 rare 类的曝光率

特别是在 LVIS 的设定中，rare 类对 novel 类的泛化非常关键。

(3) 使用生成模型（如 diffusion）合成 long-tail 类样本

虽然此处主要是 segmentation 提到，但对于 OVD 也是有效： 例如 OVDiff 生成 rare 类图像以缓解数据不平衡

④ Prompting Modeling（提示词建模）

也是 3.3 中列出的五大类之一：

“prompting modeling”

🔍 核心思想：

文本提示词决定了 VLM 中类别 embedding 的质量。

例如，在 CLIP 中：

  • “cat”
  • “a photo of a cat”
  • “a small animal that often stays indoors, cat”

会产生不同的类别向量。

因此 Prompt Engineering 或 Prompt Learning 可以决定 OVD 中 novel 类的识别效果。

📌 提示方法包括：

(1) 手工 prompt（prompt engineering）

使用一组固定模板，如：

  • “a photo of a {}”
  • “a picture of a {} in the wild”

研究表明：

多模板平均可以增强 CLIP 对 novel 类分类稳定性。

(2) 可学习提示（learnable prompts）

类似于 CoOp/CoCoOp 思想：

  • 为每个类别学习前缀 token
  • 或者为整体模型学习一组 unified prompts
  • 在训练阶段就优化 prompt，使其更适合检测场景

(3) 任务特定 prompt

例如：

  • “object detection context”
  • “object within bounding box”
  • “a cropped photo of {}”

能显著增强 region-level 对齐。

论文在 “visual grounding tasks” 部分也提到 prompt 在 grounding 任务中的增强作用

虽然不是专门讲 OVD，但方法相通。

🌈 优势：

  • 不需要更改检测器
  • 与任何 OVD 框架兼容
  • 信息成本低，但大幅影响文本对齐质量

⑤ Region-Text Alignment（区域文本对齐）

论文将该类方法作为最后一个方向：

“region text alignment”

这一类方法的核心是：

让 detector 的 box-level vision embedding 与 CLIP 的 text embedding 精确对齐。

不同于 distillation（全局模仿）与 region-text pre-training（额外预训练），这一类更注重直接对齐特征空间。

🔍 典型操作包括：

(1) 使用 CLIP 的 text encoder 生成类别向量（固定）

对于 base + novel 类：

  • 使用语言描述生成 embedding
  • 替代传统 detector 的分类器权重

这是 “open-vocabulary classifier” 的核心。

(2) 优化 detector 的视觉特征，使其与文本 embedding 匹配

训练中：

  • box embedding → 对齐 text embedding
  • 使用对比损失：contrastive / cosine loss
  • 直接优化，强化语义一致性

(3) Region-level contrastive learning

与 Region-text pretraining 相似，但：

  • 不一定需要 caption 数据
  • 可以使用 pseudo caption
  • 或只使用 base 类名称（弱监督）

这类方法通常不需要大量外部数据，却能显著增强文本对齐能力。

(4) 提升背景处理（background embedding）

论文专门描述背景 embedding：

背景 embedding 被设为 0 或可学习

处理背景是 region-text alignment 的关键点之一，因为：

  • 如果背景特征与文本 embedding 不够区分
  • 模型会将 novel 类错判为背景

开放词汇目标检测（OVD）的五大方法流派分别解决不同的底层问题：

方法类别 解决的问题 优势 代表思想
Distillation 将 CLIP 大语义空间注入检测器 简单有效 ViLD, OVR-CNN
Region-text Pretraining 提供 region-level 对齐能力 更适合检测 GLIP, region-caption对比学习
Balanced Data 克服 base 类过拟合 提升 novel 类泛化 V3Det / rare 类生成
Prompting 提高语言 embedding 质量 成本低提升大 learnable prompts
Region-text Alignment 特征空间精确对齐 提升分类鲁棒性 contrastive region-text matching

3.4 Open Vocabulary Segmentation（开放词汇分割）

开放词汇分割任务包含：

  • 语义分割（Semantic Segmentation）
  • 实例分割（Instance Segmentation）
  • 全景分割（Panoptic Segmentation）

论文根据技术路线对方法进行分类，主要有三大方向：

(1) 使用 VLM（如 CLIP）增强语义识别能力

核心思想：

把分割看成密集的分类任务，用 CLIP 的 text encoder + vision encoder 来做像素级/Mask 级的分类。

代表方法：

● LSeg

  • 将类别名称输入 VLM 的 text encoder 得到 Text Embeddings
  • 将图像通过 VLM 的视觉编码器得到 Dense Visual Features
  • 对齐两者，实现“任意类别名称”的语义分割

本质就是： 像素特征 · 文本特征 = 类别相似度 从而可以对训练中从未出现的类进行分割。

● Fusioner

  • 在 Transformer 的早期阶段使用 self-attention 融合视觉与语言特征
  • 使模型更早获得跨模态信息

● ZegFormer

  • 把任务拆成：

    • 类无关 segmentation（只找 mask，不分类）
    • mask 分类（用 CLIP 文本 embedding）

  • 非常贴合开放词汇分割的思路：分开“找东西”和“叫什么”。

● SAN / CAT-Seg / OPSNet / MaskCLIP / TagCLIP 等

这些方法共同点：

  • 在 CLIP 的基础上加轻量网络或新模块
  • 通过 mask attention、modulation、token filtering 等技术增强 VLM 的开放词汇泛化能力
  • 进一步提高 novel class 的分割性能

(2) 利用 Caption（图像描述文本）进行弱监督学习

核心思想：

Caption 里常包含训练集中没有的类别名称，因此可以作为弱监督信号，让模型学习到 novel class。

代表方法：

● OpenSeg

  • 对图像区域与 caption 中的名词使用 region-word grounding loss
  • 直接从描述文本中学习新类（例：caption 写了 “a vase on the table”，模型就能学到 vase）

属于典型的“用自然语言扩展视觉词汇”的方法。

(3) 多任务联合训练、使用检测数据辅助分割

代表方法：

● X-Decoder / FreeSeg

  • 一个模型统一处理三种分割任务：

    • Semantic
    • Instance
    • Panoptic

  • 使用可学习 prompt，实现任务级别的引导

● OpenSeeD / OpenSD

  • 同时利用检测（bbox）与分割（mask）数据训练
  • 通过 decoupled decoder 分离前景和背景
  • 提升开放词汇分割的泛化能力

(4) 使用扩散模型（Diffusion Models）

两条路线：

● 利用扩散模型中间特征（ODISE）

  • 扩散模型的中间表示自然带有语义结构
  • 在此基础上训练 segmentation decoder
  • 强大的 zero-shot segmentation 能力

● 扩散模型生成图像/Mask（OVDiff）

  • 为每个类生成 prototype images
  • 测试时用输入图像与 prototype 比较 → 最近邻分类
  • 无需训练（zero-shot）

3.5 Open Vocabulary Video Understanding（开放词汇视频理解）

视频任务包含：

  • 视频分类（Action Recognition）
  • 多目标跟踪（MOT）
  • 视频实例分割（VIS）
  • 视频全景分割（Video Panoptic）

视频开放词汇任务难点在于： 需要同时理解：时间信息 + 新类别名称 + 多物体关系

(1) 视频分类（Video Classification）

● ActionCLIP（第一篇把 CLIP 引入视频任务）

  • 使用 CLIP 的视觉编码器
  • 加一个 temporal pooling 层
  • 实现 zero-shot 视频动作识别

● I-VL / EVL / X-CLIP 等

共同点：

  • 冻结 CLIP image encoder
  • 只在 temporal fusion 上做改动（transformer、pooling 等）
  • 让模型跨帧聚合信息

● MOV（多模态视频理解）

  • 融合音频 + 视频 + CLIP
  • 扩展开放词汇能力

● Open-VCLIP / AIM / ASU

  • 探索 prompt tuning、语义单元（如 arms/head）等细粒度语言监督
  • 进一步增强 zero-shot 动作识别

(2) 开放词汇多目标跟踪（OV-MOT）

● OVTrack（第一篇开放词汇 MOT）

  • RPN 提取 RoI
  • CLIP distillation 作为类间语义对齐
  • 解决 novel classes 的跟踪问题

(3) 开放词汇视频实例分割（OV-VIS）

● MindVLT（第一篇 OV-VIS）

  • 用冻结的 CLIP backbone
  • 增加 open vocabulary classifier
  • 构建大型 VIS dataset

● OpenVIS

  • class-agnostic mask → crop 原图 → 用 CLIP 做分类
  • 与 MindVLT 不同，为两阶段方法

(4) 通用视频分割（DVIS++）

  • 同时支持：video semantic segmentation / instance segmentation / panoptic segmentation
  • 使用 mask pooling + CLIP backbone
  • 扩展开放词汇能力

3.6 Open Vocabulary 3D Scene Understanding（开放词汇 3D 场景理解）

3D 的挑战：

3D 数据标注成本极高（点云比图像更难标） 3D 训练数据远少于 2D → 无法像 CLIP 那样做大规模对比学习

因此，几乎所有方法都绕不开一个问题：

如何把 2D 的 VLM 知识迁移到 3D 模型里？

(1) 3D 识别（Classification & Segmentation）

● PointCLIP（第一篇 OV-3D）

  • 把点云投影成深度图
  • 用 CLIP-vision 处理
  • 通过 text encoder 做 zero-shot 分类

缺点：深度图损失了大量几何信息。

● PointCLIPV2

  • 改进深度图投影（Realistic Shape Projection）
  • 使用 LLM（GPT-3）生成 prompts
  • 无监督条件下大幅提升性能

● ULIP（统一三模态：点云 + 图像 + 文本）

  • 构建 point-image-text triplets
  • 通过蒸馏使 3D Encoder 对齐 CLIP 的视觉空间
  • 实现原生 open vocabulary 能力

● CLIP2 / OpenShape

  • 收集大规模三模态（图像/点云/文本）数据
  • 推动 3D open vocabulary 识别性能进一步提升

(2) 3D 开放词汇目标检测（OV-3D-OD）

● OV-3DETIC / OV-3DET

  • 解耦 3D 检测为：

    • Localization（定位）
    • Recognition（语义分类）

  • 从 2D 检测器获取 bbox 作为 prior

  • 再在 3D 空间做 refinement

● CODA / L3Det（改进 3DET）

  • 利用 2D 语义先验 + 3D 几何先验
  • 构建跨模态 alignment 模块
  • 不依赖 2D 检测器，使模型更“原生 3D”

3.7 Closely Related Tasks（相关但不同的任务）

这里总结三类容易与 Open Vocabulary 混淆的任务：

(1) Open World Object Detection（开放世界目标检测）

  • 模型需要发现未知类（unknown）
  • 之后逐步给未知类打标签、增量学习
  • 代表方法：OWOD、OW-DETR、PROB 等

区别于 OVD：

OWOD 要检测 unknown，但不要求分类成具体类别

(2) Class-Agnostic Segmentation（类无关分割）

  • 只做“分割”，不分类
  • 类似 SAM（Segment Anything）的早期思想

与 OV segmentation 区别：

OV segmentation = 分割 + 开放词汇分类 Class-agnostic = 只分割，不命名类别

(3) Open-Set Panoptic Segmentation（OSPS）

  • 对未知类标记为 unknown
  • “不需要对 novel classes 分类”

与 open vocabulary panoptic 不同：

OSPS 不需要给 novel class 具体标签，只标成 unknown。

总结（非常重要）

小节 核心问题 技术路线
3.4 分割 如何对未见类别做像素/Mask 分类 VLM + captions + multi-task + diffusion
3.5 视频理解 如何让视频模型理解新类别 CLIP + temporal fusion + tracking + VIS
3.6 3D 理解 3D 点云缺少文本/图像配对数据 投影到 2D + 三模态对齐 + 2D→3D 蒸馏
3.7 相关任务 与 OV 概念界定 OWOD / Class-Agnostic / OSPS

这些小节共同构成了“开放词汇视觉”体系： 目标是让模型在真实世界的任意、未见类别上保持稳健的识别能力。

4 Challenges & Outlook

4.1 Challenges（挑战）

论文指出开放词汇学习目前面临四类核心问题：

挑战 1：Base Classes Over-fitting（基类过拟合）

问题的本质：

  • 模型训练主要依赖 base classes 的标注（bbox/mask）
  • novel classes（未见类）没有标注
  • 因此模型学到的视觉特征更偏向 base classes

导致的问题：

  • 当 novel class 和 base class 形状或语义相似时，模型会高度偏向 base class
  • novel class 分类准确率下降
  • 模型缺乏细粒度辨别能力（如“woman vs. person”，“tower vs. building”）

论文建议方向：

  • 引入 fine-grained feature modeling（细粒度特征建模）：

    • 例如 parts（部件）
    • attributes（属性）
    • 局部形状特征

  • 让模型对 novel classes 具备更细节化的判别能力

简单理解：模型太相信训练类，需要更强的“跨类泛化能力”。

挑战 2：Training Costs（训练成本高）

开放词汇模型一般依赖：

  • VLM（如 CLIP、ALIGN）的巨大训练数据
  • 大规模 detection/segmentation 预训练
  • 图文多模态对比学习

这些训练非常昂贵。

问题：

  • 普通研究团队无法复现 VLM 级别的训练
  • 模型训练难以规模化

现有 workaround：

  • Frozen backbone（冻结骨干网络）
    • 只微调小模块（adapter/head）
    • 能节省训练成本
    • 但会降低模型容量（性能瓶颈）

核心矛盾：高性能需要大模型 → 大模型训练成本极高。

挑战 3：Cross-Dataset Training & Evaluation（跨数据集通用性不佳）

当前方法普遍存在：

  • 在 特定数据集 上表现优秀
  • 但在 另一个数据集 迁移时性能下降严重

尤其体现为：

  • LVIS、COCO、V3Det 等之间迁移困难
  • 统一模型（如 OpenSeeD）性能仍落后 dataset-specific 专用模型

深层原因：

  • 数据集类别分布差异大（long-tail、open-world gap）
  • 每个数据集的视觉风格不同
  • 标注粒度不一致

开放词汇应该“见什么都能认”，但现实中模型依然很“数据集绑定”。

挑战 4：Benchmarks & Metrics（基准评测不足）

目前评测存在问题：

1. 类别语义有重叠

如：

  • building vs. tower
  • person vs. woman

现有指标无法正确反映语义间的“层级关系/概念重叠”。

2. 数据集太小，不够“开放词汇”

论文明确指出：

真正的 open-vocabulary 应该接近现实世界的数十万级标签空间。

但现有数据集最多：

  • LVIS: 1k+ 类
  • V3Det: ~7k 类
  • 都远远不够

3. 需要新指标

比如：

  • 语义层级度量
  • 跨词汇泛化指标
  • long-tail robustness

评测体系不完备，会误导模型设计方向。

4.2 Future Work（未来方向）

论文提出五个主要研究方向：

未来方向 1：利用时间信息（Temporal Information）

视频在真实场景中占很大比例，例如：

  • 自动驾驶
  • 监控
  • 短视频理解

但目前开放词汇视频模型还很初级。

问题：

  • 现有数据集过于简单
  • Clip 数量少
  • 视频中 novel object tracking、VIS 等能力很弱

方向：

  • 更复杂的视频 dataset
  • 在视频中使用时序一致性约束
  • 研究跨帧跨类别的语义建模

总结：扩展至视频是开放词汇领域的下一块核心战场。

未来方向 2：Open-Vocabulary 3D Scene Understanding

3D 数据标注极贵，因此开放词汇更有价值。

现状：

  • 多数方法依赖将 3D 投影到 2D，再用 CLIP
  • 真正的“原生 3D VLM”几乎不存在

方向：

  • 如何让 VLM 的知识更有效迁移到纯 3D backbone
  • 跨模态对齐（image → point cloud → text）
  • 设计原生的 3D-LLM/VLM

未来方向 3：为特定任务设计 Foundation Model Adapters

例如：

  • 医学图像
  • 遥感图像

这些领域与普通图像差异巨大，CLIP 的知识无法直接泛化。

方向：

  • 设计 task-specific adapters（适配层）
  • 充分利用 VLM 的已有知识
  • 甚至结合 in-context learning（上下文提示能力）

未来方向 4：结合 Incremental Learning（增量学习）

现实世界中：

  • 新类别会持续出现
  • 但模型在学习新类时容易 遗忘旧类（catastrophic forgetting）

当前 open-world object detection 只解决：

  • 新类定位（localization） 但无法：
  • 新类分类 + 不遗忘旧类

未来需要：

  • 结合开放词汇 + 增量学习
  • 解决“持续学习 + 新类别识别”一体化的问题

未来方向 5：结合大语言模型（LLMs）

LLMs 拥有比 CLIP 更广泛的词汇知识，例如：

  • GPT-4
  • LLaMA 系列

它们包含数十万级的语义概念和常识知识。

未来方向：

  • 如何让视觉模型借用 LLM 的超大词汇？
  • 如何对齐 LLM 与 VLM 的 embedding spaces？
  • 如何利用 LLM 的推理能力提升 zero-shot 性能？

这是被普遍认为的“下一代开放词汇系统”的核心方向。

最终目标：让视觉模型与语言模型共享世界知识。

最终总结：开放词汇领域的未来趋势

从第 4 章可以看出：

1. 模型要避免过拟合到训练类

→ 需要 fine-grained learning、更强泛化能力

2. 成本太高，需要更轻量训练策略

→ frozen backbone、小模块微调

3. 模型要能跨数据集工作

→ 真正的 open-world 视觉

4. 评测体系要升级

→ 更大 vocab、语义层级指标

5. 五大研究前沿

  • 视频 + 时序
  • 原生 3D 场景理解
  • Task-specific adapters
  • 增量学习结合开放词汇
  • 视觉模型与 LLM 的深度融合

本章实际上为整个 open vocabulary 领域描绘了未来 5 年的发展路线。
