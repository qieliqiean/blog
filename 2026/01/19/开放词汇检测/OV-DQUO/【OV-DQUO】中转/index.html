下面给你一版 更偏“强自动化（脚本 + 状态机）” 的论文讲解 Skill V0（可直接落地到 Claude Code 的 .claude/skills/）。核心思想是：

  • Claude 负责写作与解释（自然语言、图解、梳理）
  • 脚本负责确定性/可重复的“结构化工作”： 解析 PDF → 抽取目录/图表 caption → 生成任务队列 → 维护 state.json → 自动拼装 4 个最终 md
  • 状态机强约束“每次只推进一个小节”：脚本在 state.json 里只允许一个 in_progress，没 done 之前 next 永远不会跳到下一节。

这套设计完全符合 Claude Skills 官方推荐的方式：把“复杂逻辑写成脚本并执行”，脚本可以 执行但不需要读入上下文，只有脚本输出会占用 token；也推荐把大内容拆到辅助文件（progressive disclosure），并用 allowed-tools 控制工具范围。(Claude Code)
